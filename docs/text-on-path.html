Exploring UX design of a text-on-path feature for mobile devices

Background

Text on Path is a popular and useful functionality in photo editing software. It lets you control the path of the text. This feature is missing in mobile editing apps.
‍
In this project, I looked into new user experiences that are easy to use on-the-go. After researching the subject, I created 3 main concepts.

Text on path example on Photoshop

Concept 1: Drawing the path

Drawing the path using your finger while swiping it on the screen.

This is a Figma file, you can pan and zoom Inside of it.

Pros
Easy to use.
‍
Intuitive.
‍
Cons
Difficult to control, hard to create a precise result.
Not editable, user needs to redraw from the beginning.

Concept 2: Text Path Presets

Choose a path preset to create the path you wish. Click the back button to finish editing.

Pros
Easy to use.
Real time text preview.
Fast. A desired result can be achieved in one click.
Cons
Limited. Not customizable.
;

‍

Concept 3: Dragging the text path

Use circle shaped anchors to adjust the text path. There will be a slider giving the user a range between 3 to 7 control points. Dragging each anchor will adjust the look of the text. Use the undo/redo buttons in case of mistakes.

This is a Figma file, you can pan and zoom Inside of it.

Pros
Intuitive.
Easy to use.
Customizable.
Real time text preview.
Cons
Finger may hide the text while editing.
;
;
;

Final Thoughts

After researching the subject and designing the wireframes, I realized that combining the last two concepts would provide the best of both worlds: Preset options and full customization through circle shaped anchors.

Pros
Easy to use.
Customizable.
Intuitive.
A desired result can be achieved in one click.
Real time text preview.
Cons
Finger may hide the text while editing.

;
;
