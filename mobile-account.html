Mobile Account

GlassesUSA

Overview

GlassesUSA.com is an online glasses and contact lenses retailer, with over 3,000,000 monthly users.

“My Account” is a place on the website where you can control and view personal information, get updates, manage subscription plans and find help and other useful information. “My Account” is an integral part of the product’s experience and has a significant influence on the whole user shopping experience. This is often a main factor in whether the user will return in the future.

GlassesUSA’s “My Account” was previously in a horrendous state - missing key features, an unsightly UI, and a bad experience overall, both on desktop and mobile devices. Because of that, it was deemed necessary to design the mobile version from scratch, and not base it on the current version.

How I worked

The whole project was handled by me. I worked with a Product Manager who gave me full control over decisions and executions. The work on the project included UX Research, Wireframing, Prototyping and Interface design.

The whole process was done by following the method of “Design Thinking”.

Empathize and define

According to the “Design Thinking” methodology, I’ve started with the first stage - Empathize.
In order to do that I’ve done a few steps:

  • Talked with the customer support team and checked customer support data from calls, chats and emails.
  • Looked at the old “My Account” mobile and desktop data
  • Made observations on users using the old “My Account”.
  • Interviewed multiple users
  • Ran Usability Tests.
  • Condacted a thorough competitor analysis.

Afterwards, I started working on the second step - “Define”. I analyzed the information from the first step and came up with guidelines that served as the bases for the whole project.

Dynamic cards

The “Ideate” step included creating a clear plan to implement the aforementioned points. One of the ideas I came up with was implementing “Dynamic Cards” that are located at the top of the “My Account” page and display the most relevant information for the user. This solution works like magic, and is similar to “Siri Suggestions” on iOS, which primarily focuses on important, relevant and urgent information.

Most users access “My Account” in order to view information about an active order, therefore I placed an informative “Dynamic Card'' when an order is active. This card contains the order status and updates at the very top. Another example - a user who bought glasses and is missing prescriptions, thus preventing the creation of the item, will receive a “Dynamic Card” reminding them to upload a valid prescription. Other instances include payment issues, expiring prescriptions, reordering contacts etc.

Finding missing revenue opportunities

While working on the project I came across (and carried through) opportunities where certain features could be implemented in a way that could increase the potential revenue.
For example, providing an option to extend the frame protection plan when it’s about to expire, and recommending an option to do an online paid eye exam when a prescription is outdated.
